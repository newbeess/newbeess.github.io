这道题目是LeetCode上的第53题－ Maximum Subarray

Find the contiguous subarray within an array (containing at least one number) which has the largest sum.

For example, given the array [−2,1,−3,4,−1,2,1,−5,4],
the contiguous subarray [4,−1,2,1] has the largest sum = 6.

分析： 算法题目的最终目的是给出满足空间复杂度下限的算法，下面我从暴力枚举一步一步优化，达到时间复杂度的下限。

  1. 暴力枚举 O(n^3)
  2. 暴力枚举的优化 O(n^2)
  3. 贪心法 O(n)

一 暴力枚举

暴力枚举在不计时间代价的情况下几乎是一种万能的方法，它将所有可能的情况全部检查一遍通过比较给出结果。拿本题来说，寻找子数组的最大和，我们可以将所有的子数组的和枚举一遍通过比较得到最大的子数组的和。

时间复杂度：O(n^3)
附加空间复杂度：O(1)

二 暴力枚举的优化

对于所有的的算法题来说，O(n^3)是难以接受的。对于算法的优化来讲，本质就是不断的去除冗余的计算。在上面的方法中，当我们计算区间［i，j＋1］的数组和＝［i，j］的数组和＋nums［j+1］，所以我们找到了存在冗余计算的部分就是求和阶段。为了去除这部分冗余，我们在i确定的时候初始化sum，所有以i为左端点的子数组可以利用前一个子数组的和。

时间复杂度：O(n^2)
附加空间复杂度：O(1)

三 暴力枚举的优化再优化

上面两步，我们枚举了所有的子数组和，实际上，如果子数组A的和<0那么以该数组开头的所有子数组都不可能是最终的结果，所以没必要枚举这种。

class Solution {
public:
    int maxSubArray(vector<int>& nums) {
        int n=nums.size();
        int res=-10000;
        int sum=0;
        for(int i=0;i<n;i++){
            sum=sum+nums[i];
            if(res<sum){
                res=sum;
            }
            if(sum<0){
                sum=0;
            }
        }
        return res;
    }
};

时间复杂度：O(n)
附加空间复杂度：O(1)
